## Description

Missing validation results in undefined behavior in `SparseTensorDenseAdd` in TensorFlow (CVE-2022-29206)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, the implementation of `tf.raw_ops.SparseTensorDenseAdd` does not fully validate the input arguments. In this case, a reference gets bound to a `nullptr` during kernel execution. This is undefined behavior. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: CVE-2022-29206 describes a TensorFlow vulnerability in SparseTensorDenseAdd due to missing input validation causing undefined behavior (NULL pointer dereference) in several older TensorFlow versions; it has a patch in newer releases. TensorFlow is a core AI framework used in AI model training and deployment, so this vulnerability pertains to the software supply chain of general-purpose AI systems. It is a security vulnerability with local access characteristics and availability impact, supported by CVE/NVD references.
