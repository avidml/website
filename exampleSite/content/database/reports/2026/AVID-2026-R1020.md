## Description

Missing validation causes denial of service in TensorFlow via `UnsortedSegmentJoin` (CVE-2022-29197)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, the implementation of `tf.raw_ops.UnsortedSegmentJoin` does not fully validate the input arguments. This results in a `CHECK`-failure which can be used to trigger a denial of service attack. The code assumes `num_segments` is a scalar but there is no validation for this before accessing its value. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: CVE-2022-29197 describes a vulnerability in TensorFlow (an AI/ML framework) that allows denial of service due to missing input validation in tf.raw_ops.UnsortedSegmentJoin. TensorFlow is a core component used to build/train/deploy AI systems, so this affects the AI software stack and the broader supply chain for general-purpose AI. The report provides affected versions and patches, satisfying evidence criteria.
