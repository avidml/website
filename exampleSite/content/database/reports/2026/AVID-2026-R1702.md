## Description

GHSL-2025-020_Retrieval-based-Voice-Conversion-WebUI (CVE-2025-43850)

## Details

Retrieval-based-Voice-Conversion-WebUI is a voice changing framework based on VITS. Versions 2.2.231006 and prior are vulnerable to unsafe deserialization. The ckpt_dir variable takes user input (e.g. a path to a model) and passes it to the change_info function in export.py, which uses it to load the model on that path with torch.load, which can lead to unsafe deserialization and remote code execution. As of time of publication, no known patches exist.

Reason for inclusion in AVID: CVE-2025-43850 describes unsafe deserialization in Retrieval-based-Voice-Conversion-WebUI, enabling remote code execution when loading a user-supplied model path via torch.load. This targets an AI software stack (voice-conversion framework) and its deployment tooling, representing a software supply-chain risk for general-purpose AI systems. The vulnerability is CVE-style (RCE) with explicit details and references, providing sufficient evidence for AVID curation.
