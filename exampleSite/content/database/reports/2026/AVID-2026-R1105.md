## Description

Null dereference on MLIR on empty function attributes in TensorFlow (CVE-2022-36011)

## Details

TensorFlow is an open source platform for machine learning. When `mlir::tfg::ConvertGenericFunctionToFunctionDef` is given empty function attributes, it gives a null dereference. We have patched the issue in GitHub commit 1cf45b831eeb0cab8655c9c7c5d06ec6f45fc41b. The fix will be included in TensorFlow 2.10.0. We will also cherrypick this commit on TensorFlow 2.9.1, TensorFlow 2.8.1, and TensorFlow 2.7.2, as these are also affected and still in supported range. There are no known workarounds for this issue.

Reason for inclusion in AVID: The CVE describes a null pointer dereference in TensorFlow's MLIR handling, causing a crash (availability impact). TensorFlow is a core AI framework, and the issue affects software components used to build, train, deploy, or run AI systems, i.e., the AI software supply chain. It is a security/vulnerability event with CVE reference and actionable patch information. Sufficient signals are present to warrant AVID curation.
