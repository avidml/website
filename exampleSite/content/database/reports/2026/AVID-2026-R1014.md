## Description

Missing validation causes denial of service via `GetSessionTensor` in TensorFlow (CVE-2022-29191)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, the implementation of `tf.raw_ops.GetSessionTensor` does not fully validate the input arguments. This results in a `CHECK`-failure which can be used to trigger a denial of service attack. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: TensorFlow is a core AI framework; CVE-2022-29191 describes a security vulnerability (denial of service due to improper input validation) in tf.raw_ops.GetSessionTensor. It affects widely used AI software packages and is in the software supply chain (dependency) used to build/train/deploy AI systems. The report provides explicit CVE details, affected versions, and references, supporting its relevance to AI stacks and supply-chain risk.
