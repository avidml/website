## Description

`CHECK` failures in `AvgPool3DGrad` in TensorFlow (CVE-2022-35959)

## Details

TensorFlow is an open source platform for machine learning. The implementation of `AvgPool3DGradOp` does not fully validate the input `orig_input_shape`. This results in an overflow that results in a `CHECK` failure which can be used to trigger a denial of service attack. We have patched the issue in GitHub commit 9178ac9d6389bdc54638ab913ea0e419234d14eb. The fix will be included in TensorFlow 2.10.0. We will also cherrypick this commit on TensorFlow 2.9.1, TensorFlow 2.8.1, and TensorFlow 2.7.2, as these are also affected and still in supported range. There are no known workarounds for this issue.

Reason for inclusion in AVID: This AVID candidate corresponds to a vulnerability in TensorFlow (AvgPool3DGrad input validation causing a denial of service). It affects a widely used AI framework, with patched commits and affected version ranges, indicating a software supply chain vulnerability in AI software stacks. The issue is explicitly about a security flaw (DoS) in a component used to build/train/deploy AI systems, satisfying criteria for AI relevance, supply-chain scope, and security impact. The report provides sufficient evidence (CVE entry, affected versions, patch info, references).
