## Description

Type confusion leading to `CHECK`-failure based denial of service in TensorFlow (CVE-2022-29209)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, the macros that TensorFlow uses for writing assertions (e.g., `CHECK_LT`, `CHECK_GT`, etc.) have an incorrect logic when comparing `size_t` and `int` values. Due to type conversion rules, several of the macros would trigger incorrectly. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: CVE-2022-29209 describes a type confusion in TensorFlow macros that can trigger a denial of service. TensorFlow is a core AI framework used in ML pipelines, training, serving, and deployment, making this a software supply-chain vulnerability within AI stacks. The issue is a security vulnerability (DoS) within software used to build AI systems, and the report provides sufficient evidence (CVE entry, advisories, and release notes).
