## Description

Missing validation causes denial of service in TensorFlow via `LoadAndRemapMatrix` (CVE-2022-29199)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, the implementation of `tf.raw_ops.LoadAndRemapMatrix does not fully validate the input arguments. This results in a `CHECK`-failure which can be used to trigger a denial of service attack. The code assumes `initializing_values` is a vector but there is no validation for this before accessing its value. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: CVE-2022-29199 describes a missing input validation in TensorFlow's LoadAndRemapMatrix causing a denial-of-service via a CHECK failure. TensorFlow is a core ML framework used in AI model development and deployment, so this is directly within the software supply chain for general-purpose AI systems. The vulnerability is a security/safety issue (DoS) with clear exploit behavior and comes with CVE details and patch references.
