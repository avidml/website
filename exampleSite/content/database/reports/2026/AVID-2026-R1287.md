## Description

Apache Airflow missing fix for CVE-2023-40611 in 2.7.1 (DAG run broken access) (CVE-2023-47037)

## Details

We failed to apply CVE-2023-40611 in 2.7.1 and this vulnerability was marked as fixed then. 

Apache Airflow, versions before 2.7.3, is affected by a vulnerability that allows authenticated and DAG-view authorized Users to modify some DAG run detail values when submitting notes. This could have them alter details such as configuration parameters, start date, etc. 

Users should upgrade to version 2.7.3 or later which has removed the vulnerability.

Reason for inclusion in AVID: CVE-2023-47037 describes an authentication/authorization vulnerability in Apache Airflow (a widely used orchestration tool) that allows authorized users to modify DAG run details. Airflow is a component commonly used to build and run ML/data pipelines, making it a software supply-chain element for general-purpose AI systems. The issue is a security vulnerability (CWE-863) affecting deployment/runtime, with a fix available in 2.7.3 or later. The report provides clear evidence and references (CVE entry, PR, OSS lists).
