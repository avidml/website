## Description

Core dump when loading TFLite models with quantization in TensorFlow (CVE-2022-29212)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, certain TFLite models that were created using TFLite model converter would crash when loaded in the TFLite interpreter. The culprit is that during quantization the scale of values could be greater than 1 but code was always assuming sub-unit scaling. Thus, since code was calling `QuantizeMultiplierSmallerThanOneExp`, the `TFLITE_CHECK_LT` assertion would trigger and abort the process. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: CVE-2022-29212 describes a core dump (availability impact) when loading TFLite models with quantization in TensorFlow, before patches were released. It is a software vulnerability in a widely used AI framework that directly affects AI model loading and execution, thus impacting general-purpose AI system stacks. The affected components are TensorFlow/TFLite runtime, a core dependency in AI pipelines, serving, and deployment. Evidence includes CVE details, affected versions, and references to fixes. Therefore it satisfies AI-related, GP AI supply chain, and security criteria with sufficient evidence.
