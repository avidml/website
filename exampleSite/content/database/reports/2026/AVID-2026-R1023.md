## Description

Missing validation causes denial of service in TensorFlow via `LSTMBlockCell` (CVE-2022-29200)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, the implementation of `tf.raw_ops.LSTMBlockCell` does not fully validate the input arguments. This results in a `CHECK`-failure which can be used to trigger a denial of service attack. The code does not validate the ranks of any of the arguments to this API call. This results in `CHECK`-failures when the elements of the tensor are accessed. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: CVE-2022-29200 describes a denial-of-service vulnerability in TensorFlow's LSTMBlockCell due to improper input validation. TensorFlow is a core ML framework used to build, train, deploy, and serve general-purpose AI systems; this is a software supply-chain vulnerability within the AI stack. The flaw is a software vulnerability (CWE-20) with a CVSS indicating available impact (availability impact) and local access requirements. The report provides explicit details and references, enabling curation.
