## Description

Missing validation causes denial of service in TensorFlow via `StagePeek` (CVE-2022-29195)

## Details

TensorFlow is an open source platform for machine learning. Prior to versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4, the implementation of `tf.raw_ops.StagePeek` does not fully validate the input arguments. This results in a `CHECK`-failure which can be used to trigger a denial of service attack. The code assumes `index` is a scalar but there is no validation for this before accessing its value. Versions 2.9.0, 2.8.1, 2.7.2, and 2.6.4 contain a patch for this issue.

Reason for inclusion in AVID: CVE-2022-29195 describes a TensorFlow vulnerability in tf.raw_ops.StagePeek due to improper input validation, causing denial of service. TensorFlow is a core ML framework used in AI pipelines, so this is a software supply-chain issue within AI software stacks. It is a CVE-style security vulnerability with clear impact (availability) and references/patches are provided. This satisfies the criteria for an AVID entry related to AI systems and their software dependencies.
