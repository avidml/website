## Description

Integer division by 0 in `tf.raw_ops.AllToAll` (CVE-2021-41218)

## Details

TensorFlow is an open source platform for machine learning. In affected versions the shape inference code for `AllToAll` can be made to execute a division by 0. This occurs whenever the `split_count` argument is 0. The fix will be included in TensorFlow 2.7.0. We will also cherrypick this commit on TensorFlow 2.6.1, TensorFlow 2.5.2, and TensorFlow 2.4.4, as these are also affected and still in supported range.

Reason for inclusion in AVID: The report describes CVE-2021-41218, a vulnerability in TensorFlow's AllToAll shape inference (divide-by-zero). TensorFlow is a core ML framework used to build/train/deploy AI systems, so this is a software supply-chain issue within AI software stacks. It is a CVE-style security vulnerability with defined affected versions and a fix, satisfying evidence for AVID curation.
